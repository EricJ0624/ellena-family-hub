# 공지 배너 미표시 원인 및 수정 (2026-03)

## 현상
- 그동안 일반 멤버 공지·관리자 전용 공지 모두 배너에 잘 나오다가, **둘 다 배너에 안 나오는** 상태가 됨.

## 원인이 된 수정

### 1. 커밋 흐름
1. **26a2fca** – 대시보드 타이틀 보강 + 공지 배너 레이아웃 개선 (minWidth: 0 등)
2. **32fe07f** – **공지 배너 단어 중간 잘림 방지**: 마퀴 제거 → 한 줄 정적 표시 + 말줄임 + N초마다 순환
3. **ac6e00e** – **revert**: 공지 배너를 “마퀴 제거 전(26a2fca) 상태”로 복원 → **다시 마퀴 버전으로 되돌림**

### 2. revert가 진행된 이유
- **32fe07f**에서 마퀴를 제거한 이유: **단어 중간 잘림**(예: "Welcome" → "lcome")을 막기 위해, 한 줄 정적 표시 + 말줄임 + 순환으로 바꿨음.
- **ac6e00e** revert는 “마퀴 제거 전 상태로 복원”이 목적이었고, 그 결과 **마퀴가 다시 켜진 버전**이 현재 코드가 됨.

### 3. 배너가 안 보이게 된 직접 원인
- **revert로 복원된 마퀴 버전**과 **32fe07f(마퀴 제거 버전)** 모두, 배너는 **“읽지 않은 공지(unread)만** 표시하도록 되어 있었음:
  - `unreadAnnouncements = announcements.filter(a => !a.is_read)`
  - `displayAnnouncements = [...unreadAnnouncements, ...unreadAnnouncements]`
  - `if (!isVisible || unreadAnnouncements.length === 0) return null;`
- 따라서 **공지가 있어도 전부 읽음 처리되면** `unreadAnnouncements.length === 0`이 되어 배너가 **아예 렌더되지 않음**.
- “일반 멤버 공지 + 관리자 전용 공지 둘 다 잘 나오다가, 지금은 둘 다 안 나온다”는,  
  **데이터가 없어서가 아니라 “전부 읽음”이거나, 특정 조건에서 unread가 0이 되는 동작**과 맞아떨어짐.

## 원래 로직 (유지)
- **읽은 공지는 배너에 안 나오는 것이 정상**입니다. 미읽음 공지만 배너에 표시합니다.
- 미읽음이 0건이면 배너를 그리지 않습니다 (`unreadAnnouncements.length === 0` 시 `return null`).
- 위 로직은 그대로 두고, 코드만 원래 의도대로 정리해 두었습니다.
